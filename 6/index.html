  1. Pada pertemuan ke 6 ini mempelajari tentang class diagram. Class diagram menggambarkan hubungan antar class atau hubungan antar tabel. Class diagaram disebut juga jenis diagram struktur karena menggambarkan apa yang harus ada dalam sistem yang dimodelkan dengan berbagai komponen. Class diagaram juga terdiri dari : Nama class, Attribut, dan Method. Nama class atau nama tabel pada class diagram. Attribut biasanya diisi dengan field yang menjadi key, nama field, tipe data, panjang data. Method apa saja yang dapat dilakukan / dialami oleh suatu objek, misalnya : select(baca),insert(tambah/simpan),update(ubah),delete(hapus),count(jumlah). Class diagram yaitu versi lengkangnya dari gabungan Entity Relationship Diagram (ERD) dan Logcal REcord Structure (LRS). Pada class diagram terdapat yang namanya Stereotype dan Multiplicity. Stereotype ialah menentukan attibute yang menjadi primary key maupun foreign key. sedangkan multiplicity yaitu menentukan tingkat hubungan/kardinalitas antar class melalui association(garis hubungan antar class).
